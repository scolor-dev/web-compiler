# Language support checklist

未対応機能の実装状況と受け入れ条件を追跡するチェックリストです。

## C

### 型・宣言

- [x] 配列（宣言、添字アクセス、初期化）
- [x] ポインター（`&`、`*`、ポインター演算、間接参照）
- [x] `struct` / `union` / `enum`
- [x] `typedef`
- [x] `const` / `static`
- [x] `signed` / `unsigned` / `short` / `long`
- [x] グローバル変数
- [x] 複数変数の同時宣言
- [x] キャスト

### 文・式

- [x] `switch` / `case` / `default`
- [x] `do while`
- [x] `goto` / ラベル
- [x] `sizeof`
- [x] 三項演算子 `?:`
- [x] ビット演算 `&` / `|` / `^` / `~`
- [x] ビットシフト `<<` / `>>`
- [x] 前置 `++` / `--`
- [x] カンマ式

### リテラル・プリプロセッサ

- [x] `float` / `double` と浮動小数点リテラル
- [x] 16進数・8進数・2進数リテラル
- [x] `#define`オブジェクト形式マクロ
- [x] `#include`の標準ヘッダー検証

### 標準関数

- [x] `printf`の幅・精度・主要変換指定子
- [x] `scanf`
- [x] `malloc` / `calloc` / `realloc` / `free`
- [x] 基本文字列関数
- [x] 仮想ファイルの読み書き
- [x] `main(int argc, char **argv)`

CはClang WASMでGNU C17としてコンパイルし、生成されたWASI実行ファイルをメモリ内の隔離環境で実行する。ファイルは実行ごとに破棄される仮想ファイルで、`argv[0]`は`program.wasm`となる。初回実行時はClang本体と標準ライブラリ資産の読み込みが発生する。

## JavaScript

### 構文・実行

- [x] `import` / `export`を単一ファイル実行向けに変換
- [x] 動的`import()`に明示的なローカル制約を適用
- [x] 実行時エラーの行・列マッピングを強化
- [x] `console.table`の表形式表示

単一ファイル・完全ローカルという境界を保つため、静的・動的importの参照先は`local:runtime`だけを許可する。`local:runtime`は`stdin`と隔離済み`console`を公開する。`export`は単一ファイル内の宣言として実行できる形へ変換する。

### Lint

- [x] JavaScriptパーサーによる完全な構文検査
- [x] 未定義識別子・未定義関数
- [x] 宣言・スコープの不整合
- [x] 重複宣言
- [x] 到達不能コード
- [x] 正規表現・テンプレートリテラルの正確な解析

### ホストAPI方針

以下は言語構文の未対応ではなく、完全ローカル・隔離実行を守るための制限です。

- [x] DOM APIをWorkerから分離する
- [x] Node.js固有APIを提供しない
- [x] ネットワークAPIを無効化する
- [x] `stdin`を明示的な入力境界として提供する
- [x] 実行時間制限で無限ループを停止する

## 完了条件

- [x] 各機能群に正常系とエラー系の自動テストがある
- [x] Rustテスト、フロントエンドテスト、型検査、本番WASMビルドが成功する
- [x] 対応外として残す項目は、理由と代替手段をこの文書へ明記する
